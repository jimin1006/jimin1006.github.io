챕터6
6.1. 구성
다른 페이지로 이동할 수 있는 링크
태그를 통해, 텍스트에 생명을 불어넣음(제목인지..)

markup: 생명을 불어넣는 것

태그로 감싸준다(여는 태그, 닫는 태그->범위가 특정되어야 함)

요소
<p> : 문단의 형식을 띄게 됨

속성
ex. class="memo"
속성이름="속성값"
여러 개의 속성의 경우)
한 칸 씩 띄어놓고 작성
ex. 속성이름="속성값" (공백) 속성이름="속성값"

주석: 브라우저에 뜨지 않도록 무시
<!--(여는태그) --> (닫는태그)

문서구조
boilerplate: 반복되는 구조 -> 단축키로 한 번에 생성 가능

6.2. HTML <head>
HTML 구성하는 중심 축: head, body
UTF-8: 전 세계 언어 모두 지원(반드시 설정해야)
인터넷 익스플로러(IE)=edge: 최신 버전 지원
device-width: 디바이스에 따라 너비가 달라짐 -> 폭을 장비에 맞추겠다
1.0: 100%, 확대나 축소를 하지 않겠다

6.3. 블록 요소와 인라인요소
HTML 요소: 블록 요소와 인라인 요소로 크게 나뉨
블록: 줄바꿈이 일어남, 하나의 면을 구성
인라인: 특정 글자에만 적용, 

블록 요소는 블록 요소, 인라인요소 포함 가능
인라인 요소는 블록 요소를 포함할 수 없음

6.4. HTML <body>
실제 브라우저에 보여지는 내용
제목: heading tag
제목의 위계에 따라 
여섯 계의 계층: <h1> ~ <h6> / 일반적으로 <h3>까지 주로 사용
h1: 하나의 HTML에서는 하나만 (대제목이라서)
*heading 태그는, 글씨를 키우려는 목적으로 사용하지 않는다

<p>: 하나의 p태그로 감싸 문단을 구성해준다

줄 나눔이 어떻게 되는 건가?
line break를 나타내는 br 태그를 사용해줘야 한다
 -> 단일 태그(여는 태그 닫는 태그 없이 단독으로 가능)
ex. <p> 멋쟁이사자처럼 스쿨이<br>

순서매기는 태그: ol(리스트 만드는것, ordered list), li(내부 요소 하나하나에는)
목록형식이지만 순서 안 매기는 태그: ul(리스트 만드는, unordered list), li(내부 요소)

<a>: 다른 페이지로 이동할 수 있는 링크걸기
ex. <a href="연결할링크(url)">프론트엔드스쿨</a>
*href: 주소를 참조하는 속성명
 -> 이 경우는 내 페이지를 그대로 유지하지 못함
만약에 새 창을 띄우고 싶다면?
-> target 속성 추가해서, _blank 넣어주기
ex. <a href="연결할링크(url)"> target="_blank">프론트엔드스쿨</a>

<span>태그
내가 적용하고 싶은 영역에만 <span>태그로 마크업해주면 됨
ex. 멋쟁이사자처럼 스쿨에만 적용
ex. <p><span>멋쟁이사자처럼 스쿨</span>이<br>여러분을 환영합니다</p>
보통 단독 사용x class 속성을 사용해, javascript/css의 적용대상임을 밝힘
ex. <p><span class="">멋쟁이사자처럼 스쿨</span>이<br>여러분을 환영합니다</p>

<div> 태그
여러 요소를 묶어 그룹화
span: 문자열의 일부 단위로 적용
div: HTML 요소 단위로 적용

<a href="#">: 이동 경로는 없지만 링크처럼 기능함

마우스 우클릭 -> format document : 올바르게 배치 바꿔줌
